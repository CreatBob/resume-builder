# 前端最佳实践规则沉淀

## 1. 背景

本次前端规则沉淀的目标，不是为某一次具体回归补一条“对症规则”，而是把最近暴露出的共性风险抽象为可长期复用的最佳实践约束。

当前需要补强的并不是某个单点页面行为，而是三类更上层的前端协作原则：

1. 布局系统中的滚动职责必须显式设计。
2. 样式作用域与不同渲染域之间必须清晰隔离。
3. 高风险前端改动必须通过页面事实验收，而不是仅靠静态检查。

因此需要将这些原则沉淀为仓库级前端规则，作为后续类似工作的统一约束。

## 2. 目标

将近期暴露出的共性风险抽象为长期有效的前端最佳实践，并写入最合适的仓库规则文档，使后续类似改动有统一约束可依，而不是每次按具体事故补细则。

## 3. 范围

- 明确布局容器与滚动职责的原则性约束。
- 明确样式作用域与渲染域隔离的原则性约束。
- 明确高风险前端改动的页面级验收原则。
- 只在最合适的规则文档中落规则，避免重复分散。

## 4. 非目标

- 不把本次规则沉淀扩散到与问题无关的全局规则、后端规则或代码审查产物中。
- 不把需求文档本身当成长期规则入口。
- 不新增测试代码。
